# 纯 Vimscript 高效编辑配置


一份不依赖任何插件的 `.vimrc`，用纯 Vimscript 实现了模糊搜索、文件树、Git 状态/diff、
git blame、buffer 切换栏、行级 Git 变更标记等功能。

## 设计思路

我希望有一个编辑器,而且只是编辑器,不应该用任何ide的功能来完成编辑器.所以在我的设计当中,所有的功能都只通过文本的方式来完成
我的设计只是为了能够更好的满足一个编辑器的功能,方便的把我自己在开发当中用到的基本功能放在里面


## 要求

- Vim 8.2+（需要 popup window、text property、`matchfuzzy()` 等特性）
- 可选依赖：
  - [ripgrep (rg)](https://github.com/BurntSushi/ripgrep)：用于文件名列举和内容搜索。
    没有 rg 时文件名搜索自动退化为 Vim 内置的 `globpath`，内容搜索不可用。
  - `git`：Git 相关功能需要
  - `diff`：行级变更标记需要（系统自带）

## 安装

```sh
cp .vimrc ~/.vimrc
```

`<leader>` 键设置为空格（`<Space>`）。

## 功能一览

| 快捷键 | 功能 |
|--------|------|
| `<leader>ff` | 模糊搜索项目文件名（带预览） |
| `<leader>fg` | 搜索项目内容（带预览） |
| `<leader>e` | 开关文件树侧边栏 |
| `<leader>g` | 打开 Git 状态窗口（带 split diff） |
| `<leader>b` | 底部 buffer 切换栏 |
| `<leader>d` | 预览光标处修改块的 diff |
| `<leader>p` | `:find` 光标下的单词（自动去掉斜杠） |
| `<leader>l` | 开关空白字符显示 |
| `Ctrl-s` | 保存（仅在有修改时写入） |
| `Ctrl-h/j/k/l` | 在分屏窗口之间移动 |

## 模糊搜索（`<leader>ff` / `<leader>fg`）

纯 Vimscript 实现的弹窗式模糊搜索，左侧结果列表、右侧实时预览（带语法高亮）。

- `<leader>ff`：模糊搜索项目中的文件名（`rg --files`，含隐藏文件，排除 `.git`）
- `<leader>fg`：搜索项目内容，输入至少 2 个字符后开始搜索（`rg --vimgrep`，smart-case）
- 可视模式下先选中一段文本再按 `<leader>ff` / `<leader>fg`，会把选中内容作为搜索关键词
- 文件名模糊匹配无结果时，会自动降级为「按片段相似度」匹配，
  适合粘贴带引号、行号、部分路径的文本

弹窗内按键：

| 按键 | 作用 |
|------|------|
| `Ctrl-j` / `Ctrl-k`（或 `↑` `↓`） | 上下选择 |
| 回车 | 在当前窗口打开 |
| `-` | 水平分屏打开 |
| `\` | 垂直分屏打开 |
| `Esc` / `Ctrl-c` | 关闭 |
| 退格 | 删除一个字符 |

内容搜索（`<leader>fg`）回车后会打开文件并跳转到匹配行，预览窗中匹配行高亮显示。

也可用命令形式：`:FFiles`、`:FGrep`。

## 文件树（`<leader>e`）

在左侧打开一个 30 列宽的目录树侧边栏，再按一次 `<leader>e` 关闭。
目录在前、文件在后，按名称排序（忽略大小写），排除 `.git`。

树内按键：

| 按键 | 作用 |
|------|------|
| 回车 / `o` | 打开文件，或展开/折叠目录 |
| `-` | 水平分屏打开文件 |
| `\` | 垂直分屏打开文件 |
| `r` | 刷新目录内容 |
| `q` | 关闭侧边栏 |

## Git 状态窗口（`<leader>g`）

在独立标签页中打开 Git 状态界面（需要在 git 仓库内）：

- 左侧：`git status` 变更文件列表（`M`、`A`、`??` 等状态标记）
- 中/右：两个窗格用 Vim 内置 diff 模式实时显示 split diff（旧版本 | 新版本）
  - 已暂存文件：HEAD vs 暂存区
  - 未暂存文件：暂存区 vs 工作区
  - 未跟踪文件：空 vs 文件内容

状态列表内按键：

| 按键 | 作用 |
|------|------|
| `j` / `k` | 移动选择，右侧 diff 实时更新 |
| 回车 | 打开选中的文件 |
| `s` | stage 选中文件（`git add`，未跟踪文件/目录同样适用） |
| `u` | unstage 选中文件（`git restore --staged`，老版本 git 自动退回 `git reset`） |
| `r` | 刷新状态 |
| `q` | 关闭 |

## 行级 Git 变更标记与 diff 预览

- 打开/保存文件后，以及停止编辑约 0.3 秒后自动刷新
- 行号左侧 sign 列标记当前文件相对 git 暂存区的变化：
  - 绿色 `+` = 新增行
  - 黄色 `~` = 修改行
  - 未跟踪文件的所有行都标记为新增
- 把光标放在修改块内按 `<leader>d`，弹出该修改块的完整 diff 预览
  （含上下文、diff 语法高亮），按 `q` 或 `Esc` 关闭

## 状态栏 git blame

光标停留约 1 秒（`updatetime=1000`）后，状态栏右侧显示当前行的 blame 信息：

```
 [作者名 2024-01-15 提交摘要…]
```

同一行不重复查询；未提交的修改显示 `[Not committed]`。

## Buffer 切换栏（`<leader>b`）

在底部弹出横向 buffer 栏，每个 buffer 带两个字母标识符
（绿色 = 切换过去，红色 = 关闭它），标识符按 `a b c ... z aa bb ...` 分配。

| 按键 | 作用 |
|------|------|
| `Tab` / `Shift-Tab` | 循环切换 buffer（即时生效） |
| 字母键 | 按标识符过滤；只剩唯一候选时立即执行切换/关闭 |
| 回车 | 切换到当前选中的 buffer；输入精确匹配某标识符时执行该项 |
| 退格 | 删除输入 |
| `Esc` | 关闭 |

有未保存修改的 buffer 名字后带 `+`，且不会被关闭。

## 自动补全

基于 Vim 内置补全（无需插件）：

- 进入插入模式、以及每次输入可打印字符时自动弹出补全菜单
  （`help` 和 `gitcommit` 文件类型除外）
- 补全来源：当前文件、其他 buffer、字典、标签、include 文件等
- `Ctrl-n` / `Ctrl-p` 在候选间移动
- 针对 HTML / CSS / JavaScript / TypeScript 配置了对应的 omni 补全（`Ctrl-x Ctrl-o`）

## 其他编辑体验

- **自动缩进检测**：默认 2 空格缩进；打开文件时扫描前 100 行，
  自动把 `shiftwidth`/`tabstop` 调整为文件实际使用的缩进宽度
- **空白字符显示**（`<leader>l` 开关）：Tab 显示为 `→`，
  行首/行尾空格显示为暗灰色 `·`，nbsp 显示为 `␣`
- **文件外部变化**：开启 `autoread`，光标停留、切换 buffer、窗口获得焦点时自动检查。
  磁盘和 Vim 中都有修改时弹窗询问：
  - `Y`：用 Vim 中的内容覆盖磁盘
  - `N`：放弃 Vim 修改，加载磁盘最新内容
  - `Esc`：暂不处理，下次再询问
- **搜索**：高亮搜索结果（`hlsearch`）、增量搜索（`incsearch`）
- **界面**：显示行号、每个窗口顶部显示文件名（winbar）、
  状态栏常显且标记修改状态 `[+]`、任意行数变化都报告（`report=0`）
- **路径查找**：`path` 包含 `**`，配合 `set wildmenu` 可用 `:find` 递归查找文件；
  `<leader>p` 会取光标下的单词、去掉斜杠后执行 `:find`
